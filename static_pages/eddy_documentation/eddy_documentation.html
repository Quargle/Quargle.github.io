Introduction

Eddy is a HTML output generator for MCNP and SCALE, it imports an MCNP or SCALE output file, extracts the important data, and writes it to a user-friendly HTML file.

It was created by Cerberus Nuclear, and is provided as an open-source project under the GPL.

Eddy is a purely python-based package, with results presented in a single HTML file (with CSS styling included in that file, and no javascript).

Versions

Eddy is available as a Windows executable file and as a python package which can be found on PyPI, the Python Package Index, and can be installed via pip. The source code is also available on GitHub. This documentation refers to the python package version of Eddy.

Requirements

Eddy requires Python version 3.6 or later to be installed. Eddy will install other dependencies, including the Jinja2 and importlib-resources packages.

If the source code version of Eddy is obtained from Github, these packages will need to be installed manually, and the pytest and pytese-mock packages will be required to run the unit tests.

Installation

Eddy can be installed from pip using the command

pip install eddy-mc

and accessed from within a python program by using

from eddymc import eddy eddy.main()

where eddy.main() takes two arguments; the file path of the MCNP/SCALE output file to be processed, and a scaling factor to be applied to any results

MCNP

MCNP features include:

  • Eddy can convert F2, F4, F5, F6 and F6+ tallies
  • Eddy can accept average tallies, given in the form F4:N (4 5)
  • Eddy can take an MCNP criticality output and show k-effective for the first half, second half and total calculation
  • Eddy will present a warning if an MCNP case was halted due to lost particles
  • Eddy will clearly present any FATAL ERROR messages in the MCNP output
  • Any valid HTML tags found in the MCNP output file will be escaped, preventing any unwanted HTML or javascript injected into the MCNP output from making its way into the HTML output file
  • For F2, F4 and F5 tallies, the units are presented as microSieverts per hour. The actual result from MCNP is typically multiplied by some conversion factor, so the tally output could be in any unit. The uSv/h units have been left in for now as this seems to be by far the most common conversion factor used for shielding calculations.
  • Eddy currently does not accept multi-particle tallies of the form F4:N,P
    • SCALE

    Eddy also processes certain SCALE output cases.

    Since SCALE is an extremely large program, Eddy cannot cover all SCALE cases, but should work for any MAVRIC shielding case.

    It will also work for outputs that contain both ORIGEN and MAVRIC cases, although the ORIGEN results will not be reported by Eddy.
